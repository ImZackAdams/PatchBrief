Roadmap

What PatchBrief is building and why.

This is the honest product direction: what exists now, what's planned next, what comes later, and what is not being built yet. No timeline promises.

Current

What exists now

  • Public feed of sample security briefs
  • Static site on GitHub Pages
  • Watchlist pilot signup form
  • Sample watchlist brief format
  • Manual and semi-manual brief curation
  • CISA KEV ingestion (local CLI)
  • Watchlist matching (local CLI)
  • Email brief delivery via Resend (local CLI)
Next

What's being built

  • Structured feed item publishing workflow
  • RSS feed auto-generation
  • Weekly public email newsletter
  • Saved watchlist matching for pilot users
  • Filtered brief delivery to pilot subscribers
  • NVD enrichment for feed items
  • GitHub Actions build pipeline
Later

Planned but not started

  • Paid personal watchlist tier (~$9/mo)
  • Team and MSP watchlist tiers (~$29–$99/mo)
  • Important-only alert cadence
  • Slack and Teams webhook integrations
  • Client-ready brief summaries for MSPs
  • Microsoft MSRC enrichment
  • GitHub Security Advisories integration
  • Vendor advisory ingestion
Not doing yet

Out of scope for now

  • Full web dashboard or portal
  • Environment scanning or agent installation
  • Exposure verification or reachability testing
  • Automated remediation or ticketing
  • AI SOC copilot or chatbot
  • Threat intel platform features
  • SLA or uptime guarantees

What PatchBrief is not

PatchBrief tracks public sources. It does not scan your environment, verify whether a vulnerability affects your specific systems, or replace vendor guidance.

It is a signal layer, not a security program. The briefs point to public sources and suggest checks — operators and their teams make the calls.
